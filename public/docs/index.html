Introduction

This API aims to provide all the information you need in order to work with our API

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_TOKEN}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Authentication

Endpoints for login, registration, and logout

User registration

This endpoint allows you to register a new user.

Users will not be able to create or join purchase goals if they are not registered and logged in
Example request:
curl --request POST \
    "http://localhost/api/register" \
    --header "Content-Type: multipart/form-data" \
    --header "Accept: application/json" \
    --form "name=qohsesjhpekjknvtxpgbqfeq"\
    --form "email=atillman@example.net"\
    --form "password=jcxJZ!A"\
    --form "password_confirmation=jcxJZ!A"\
    --form "address=praesentium"\
    --form "phone_number=illum"
const url = new URL(
    "http://localhost/api/register"
);

const headers = {
    "Content-Type": "multipart/form-data",
    "Accept": "application/json",
};

const body = new FormData();
body.append('name', 'qohsesjhpekjknvtxpgbqfeq');
body.append('email', 'atillman@example.net');
body.append('password', 'jcxJZ!A');
body.append('password_confirmation', 'jcxJZ!A');
body.append('address', 'praesentium');
body.append('phone_number', 'illum');

fetch(url, {
    method: "POST",
    headers,
    body,
}).then(response => response.json());

User authentication

This endpoint allows authenticate or log in a user.

An authentication token is always generated upon successful login.
Example request:
curl --request POST \
    "http://localhost/api/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"ian76@example.org\",
    \"password\": \"{_aV!Uh]\\\"nUgRH\"
}"

const url = new URL(
    "http://localhost/api/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "ian76@example.org",
    "password": "{_aV!Uh]\"nUgRH"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
