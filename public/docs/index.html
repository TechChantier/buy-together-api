Introduction

This API aims to provide all the information you need in order to work with our API

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_TOKEN}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Authentication

Endpoints for login, registration, and logout

User registration

This endpoint allows you to register a new user.

Users will not be able to create or join purchase goals if they are not registered and logged in
Example request:
curl --request POST \
    "http://localhost/api/register" \
    --header "Content-Type: multipart/form-data" \
    --header "Accept: application/json" \
    --form "name=sknonxvhbdfqioxelfzhdsfu"\
    --form "email=rosalind75@example.org"\
    --form "password=^]C"<"<!TUNt"\
    --form "address=quia"\
    --form "phone_number=illum"\
    --form "profile_pic=@C:\Users\user\AppData\Local\Temp\php24AB.tmp" 
const url = new URL(
    "http://localhost/api/register"
);

const headers = {
    "Content-Type": "multipart/form-data",
    "Accept": "application/json",
};

const body = new FormData();
body.append('name', 'sknonxvhbdfqioxelfzhdsfu');
body.append('email', 'rosalind75@example.org');
body.append('password', '^]C"<"<!TUNt');
body.append('address', 'quia');
body.append('phone_number', 'illum');
body.append('profile_pic', document.querySelector('input[name="profile_pic"]').files[0]);

fetch(url, {
    method: "POST",
    headers,
    body,
}).then(response => response.json());

User authentication

This endpoint allows authenticate or log in a user.

An authentication token is always generated upon successful login.
Example request:
curl --request POST \
    "http://localhost/api/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"odie74@example.net\",
    \"password\": \"6;?lB9x6U\\\"I]t)cfOu&1\"
}"

const url = new URL(
    "http://localhost/api/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {

};
