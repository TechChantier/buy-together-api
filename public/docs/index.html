Introduction

This API aims to provide all the information you need in order to work with our API

Base URL: https://rrn24.techchantier.site/buy-together-api/public

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_TOKEN}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Authentication

Endpoints for login, registration, and logout

User registration

This endpoint allows you to register a new user.

Users will not be able to create or join purchase goals if they are not registered and logged in
Example request:
curl --request POST \
    "http://localhost/api/register" \
    --header "Content-Type: multipart/form-data" \
    --header "Accept: application/json" \
    --form "name=fpjtbjpvqveb"\
    --form "email=raina52@example.org"\
    --form "password=e.`\UzNA.>i%](}p"\
    --form "address=quos"\
    --form "phone_number=dolor"\
    --form "profile_pic=@C:\Users\user\AppData\Local\Temp\php7B2C.tmp" 
const url = new URL(
    "http://localhost/api/register"
);

const headers = {
    "Content-Type": "multipart/form-data",
    "Accept": "application/json",
};

const body = new FormData();
body.append('name', 'fpjtbjpvqveb');
body.append('email', 'raina52@example.org');
body.append('password', 'e.`\UzNA.>i%](}p');
body.append('address', 'quos');
body.append('phone_number', 'dolor');
body.append('profile_pic', document.querySelector('input[name="profile_pic"]').files[0]);

fetch(url, {
    method: "POST",
    headers,
    body,
}).then(response => response.json());

User authentication

This endpoint allows authenticate or log in a user.

An authentication token is always generated upon successful login.
Example request:
curl --request POST \
    "http://localhost/api/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"abbie31@example.net\",
    \"password\": \"_jUj>ci\"
}"

const url = new URL(
    "http://localhost/api/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {

    "password": "_jUj>ci"
};
